# 專案檔案說明：api/submenu.php

## 檔案用途
此檔案為後台「次選單管理」的後端處理程式。它負責處理前端 `modal/submenu.php` 提交的表單數據，執行次選單的**批次更新**、**批次刪除**以及**批次新增**功能。

## 主要邏輯流程

1. **環境初始化**：透過 `include_once "db.php"` 取得資料庫操作權限及工具函式。
2. **處理現有資料 (Edit/Delete)**：
   - 遍歷 `$_POST['text']`。
   - 若 ID 存在於 `$_POST['del']` 陣列中，執行 `del()` 刪除。
   - 否則，更新該筆資料的 `text` 與 `href` 並執行 `save()`。
3. **處理新增資料 (Insert)**：
   - 遍歷 `$_POST['new_text']`。
   - 若內容不為空，則新增一筆資料，並將其 `main_id` 設為從 URL (`$_GET['main_id']`) 傳入的值，以建立主從關係。
4. **頁面跳轉**：完成後導回選單管理頁面 (`back.php?do=menu`)。

## 欄位對應
- `main_id`: 所屬主選單的 ID。
- `text`: 次選單顯示文字。
- `href`: 次選單連結位址。
- `del`: 勾選刪除的次選單 ID 陣列。

## 注意事項
- 次選單的新增與修改是合併在同一個表單中送出的。
- 連結次選單與主選單的關鍵在於 `main_id`。
